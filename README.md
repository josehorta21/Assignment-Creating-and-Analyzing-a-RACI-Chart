# Seattle Watch Repair — RACI Chart Assignment

This repository contains the RACI (Responsible, Accountable, Consulted, Informed) chart and analysis for the **Seattle Watch Repair** website project.

It was produced for a course assignment on project roles and responsibilities, following the structure and best practices in [Atlassian's RACI Chart guide](https://www.atlassian.com/work-management/project-management/raci-chart).

## Author

Jose Horta — Computer Science, North Seattle College

## About the Project

Seattle Watch Repair is an independent watch restoration shop based in Seattle (seattlewatchrepair.com). The project goal was to design and launch a full-stack web application — built with React and Vite on the frontend and Express on the backend — that showcases the shop's restoration work, communicates trust through a 1970s/80s retro-modern visual style, and reduces repetitive customer questions through an AI-powered chatbot integrated with the Anthropic API. The site is hosted on GoDaddy with Node.js and cPanel, with Apache `.htaccess` proxy configuration.

## Document Structure

The deliverable follows the four-step RACI process:

1. **Identify Tasks and Workload** — twelve tasks and subtasks across discovery, design, build, deployment, QA, and launch.
2. **Identify Roles** — six distinct roles with named owners and primary responsibilities.
3. **Build the Chart** — initial RACI matrix.
4. **Analyze and Revise** — gaps, overlaps, balance, consultation, and information-flow review, followed by a corrected final chart.

A final section describes how to embed the chart into project documentation tools such as Confluence, including which project pages should host it and why.

## Key Roles

- **PM** — Project Manager
- **CL** — Client / Business Owner
- **FE** — Frontend Developer
- **BE** — Backend & AI Developer
- **UX** — UX & Visual Designer
- **QA** — QA / Tester
